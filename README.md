# Автоматизатор Рейсов ТС (Vehicle Trip Analyzer)

Этот проект предоставляет микросервис (FastAPI) с веб-интерфейсом для автоматической обработки выгрузок системы видеонаблюдения и расчета рейсов транспортных средств. 

## Особенности
- **Парсинг выгрузок OCR**: очистка сырых данных и парсинг русских дат.
- **Исправление ошибок распознавания (Левенштейн + Временные окна)**: алгоритм склеивает похожие номера, которые проехали подряд и/или есть в вашем справочнике.
- **Мощный Excel Экспорт**: формируется отчет с детализацией каждого рейса, а также сводные таблицы по машинам и контрагентам. Ограничение знаков после запятой (max. 3).
- **Справочник `base.xlsx`**: независимое хранение базы машин. Обновляется через UI и сохраняется между деплоями в Docker Volume.
- **Drag & Drop UI**: корпоративный, легкий интерфейс без лишних деталей.

## Инструкция по развертыванию
Проект разворачивается через `docker-compose`. 

```bash
docker compose up -d --build
```

### CI/CD Pipeline
Деплой настроен через **GitHub Actions** (`.github/workflows/deploy.yml`). При пуше в ветку `main`:
1. Выполняется установка зависимостей.
2. Прогоняются автотесты `pytest`.
3. Код копируется на сервер через SCP.
4. На сервере собирается новый Docker контейнер (с сохранением Docker Volume `base_data` для справочника).

## Разработка и Тестирование
В проекте реализованы тесты на базе `pytest`. Для запуска:

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pytest tests/
```

### Обновление Справочника
Вы можете обновить `base.xlsx` на стартовом экране(Внизу слева). Файл используется для обогащения данных (Марка, Контрагент). Он будет надежно сохранен на сервере, даже если код обновится через GitHub Actions.
